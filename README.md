# Churn-Analysis


# Project Overview

Designed and implemented an end-to-end customer churn analytics project for an OTT streaming platform to identify high-risk subscribers, measure revenue impact, and provide actionable business recommendations. The project combines SQL, Python, and data visualization techniques to transform raw customer, subscription, and support data into executive-level insights.

# Key Responsibilities

Connected Python with SQLite database to extract relational data using SQL queries.
Cleaned and transformed customer, subscription, and support datasets using Pandas and NumPy.
Engineered business metrics including customer tenure, churn status, retention rate, revenue at risk, and CLTV loss.
Performed exploratory data analysis (EDA) using aggregation, group-by operations, and pivot tables.
Created visual dashboards using Matplotlib and Seaborn to identify churn trends and customer behavior.
Delivered business recommendations based on churn drivers, subscription plans, customer support quality, and regional performance.

# Business KPIs

Overall Churn Rate: 28.6%

Retention Rate: 71.4%

Monthly Contract Churn: 55.6%

Annual Contract Churn: 8.3%

Average Customer Tenure: 1,451 Days

ARPU: ₹18.8

Revenue Loss Due to Churn: ₹73.94

CLTV Lost: ₹2,047

Revenue Loss Percentage: 18%

# Business Impact

Identified Basic subscription customers as the highest churn segment.

Found Karnataka to have the highest regional churn.

Quantified monthly recurring revenue leakage and customer lifetime value erosion.

Recommended migrating monthly subscribers to annual plans, improving customer support, prioritizing high-value customers, and optimizing pricing strategies to improve customer retention.
